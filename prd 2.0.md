糖果翻牌記憶遊戲 - 對戰模式 PRD
一、專案概述
專案名稱：糖果翻牌記憶遊戲 - 對戰模式

一句話描述：一款支持兩名玩家輪流翻牌匹配糖果甜點圖標的記憶遊戲，具有連對得分機制和時間限制的對戰模式。

版本：1.0

二、問題與目標
問題陳述
現有的單人記憶遊戲缺乏社交互動和競技性，玩家只能挑戰自己的記憶極限，無法與朋友進行實時對戰，降低了遊戲的重複可玩性和社交樂趣。

目標
將傳統單人記憶遊戲轉變為互動性強的對戰遊戲

引入得分機制和連對獎勵系統，增加策略性

添加時間限制，提高遊戲緊張感和挑戰性

創建視覺吸引人的糖果主題界面，提升玩家體驗

三、目標用戶
用戶特徵
年齡：8-25歲的兒童和青少年

技術水平：熟悉基本網頁操作，能使用觸控設備或滑鼠

使用場景：課間休息、家庭聚會、朋友互動、休閒娛樂

遊戲偏好：喜歡快速、簡單、有競爭性的小遊戲

用戶需求
需要簡單易懂的遊戲規則，快速上手

希望與朋友進行實時互動和競爭

需要視覺反饋和得分激勵來保持遊戲興趣

期望遊戲有適當的挑戰性和重複可玩性

四、用戶故事
用戶故事1：作為一名學生，我想要在課間與同學進行快速記憶對戰，以便放鬆心情並鍛鍊記憶力。

用戶故事2：作為一名家長，我想要一個安全的網頁遊戲，可以和孩子一起玩，以便增進親子互動並訓練孩子的記憶能力。

用戶故事3：作為一名遊戲玩家，我想要一個有挑戰性和競爭性的記憶遊戲，以便與朋友比較得分並享受勝利樂趣。

五、功能需求與驗收標準
功能1：遊戲初始化與設置
描述：遊戲開始時，系統應隨機生成8對糖果主題圖標（共16張卡片），並以背面朝上的方式顯示在4×4的網格中。

驗收標準：

Given 玩家打開遊戲頁面

When 頁面加載完成

Then 顯示16張背面朝上的卡片，按4×4網格排列，所有卡片圖案已隨機分配

功能2：遊戲開始流程
描述：玩家點擊"開始遊戲"按鈕後，系統應展示所有卡片2.5秒，然後翻轉回背面，計時器開始倒數。

驗收標準：

Given 玩家點擊"開始遊戲"按鈕

When 按鈕被點擊

Then 所有卡片翻轉顯示2.5秒，然後翻回背面，90秒倒計時開始

功能3：玩家輪流翻牌機制
描述：兩名玩家輪流翻牌，每次翻兩張牌。如果匹配成功，該玩家得分並繼續回合；如果匹配失敗，輪到另一玩家。

驗收標準：

Given 玩家1的回合

When 玩家1點擊兩張卡片且圖案不匹配

Then 卡片翻回背面，回合切換到玩家2

功能4：連對得分系統
描述：連續匹配成功時得分翻倍（第1次得1分，第2次得4分，第3次得9分，依此類推）。

驗收標準：

Given 玩家連續匹配成功3次

When 計算第3次匹配得分

Then 玩家獲得9分（3×3）

功能5：遊戲結束判定
描述：當所有卡片匹配完成或時間用完時遊戲結束，顯示雙方得分和獲勝者。

驗收標準：

Given 所有卡片已匹配

When 最後一對卡片匹配成功

Then 顯示遊戲結束畫面，比較雙方得分並宣布獲勝者

功能6：遊戲重啟功能
描述：玩家可以隨時重新開始遊戲，重置所有狀態。

驗收標準：

Given 遊戲進行中

When 玩家點擊"重新開始"按鈕

Then 遊戲狀態重置，卡片重新隨機排列，計時器重置為90秒

六、技術約束
必須遵守
使用純HTML/CSS/JavaScript實現，無需後端服務

完全響應式設計，支持手機和桌面設備

使用Font Awesome 6.4.0圖標庫

兼容性要求
支持Chrome 60+、Firefox 55+、Safari 12+、Edge 79+

支持iOS 12+和Android 8+的移動瀏覽器

支持觸摸設備和滑鼠操作

不要做
不使用任何外部JavaScript框架（如React、Vue）

不依賴瀏覽器插件或擴展

不收集或存儲用戶個人數據

不實現複雜的網絡對戰功能（保持為本地對戰）
